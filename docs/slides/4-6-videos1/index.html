Project Videos: What are they NOT?

  • Screen recording of your software
  • Slideshow about your software

Project Videos: What are they?

  • Screen recordings + text + images
  • Narration
  • Multiple clips combined
  • A story about your software

A story about your software

Many visualization "rules of thumb" apply to making videos too:

  • Overview first, then details

    Always answer "what are we looking at" ASAP

  • Guide the viewer's attention

    Highlight what you want the viewer to look at:

    Use color, shape, movement, etc.

  • Linearity is rarely the best strategy

    "The user does this, then the user does this, then...":

    Instead, highlight the importance of what the user just did

More on storytelling next week!

Other considerations when making vis software videos

  • Try to only show one software feature / user achievement at a time

    Humans have a hard time focusing on multiple things at once

    Give visual AND auditory cues for what the viewer should be paying attention to:

    "Ocean current velocity is depicted using artist-drawn textures. Near the center of the eddy, notice how the texture changes are used to indicate rotational velocity of this eddy.
  • Keep it movin'

    Don't dwell too long on any single clip

    If possible, always have something moving on-screen

    In the narration, speak energetically

Possible timeline for a 3:30 project video

  1. Introduction [0:30]

    Show overview of your visualization

    What are we looking at?

    (very brief) background on the data if necessary

  2. Interactive Tech 1 [1:00]

    Show a user achievement with tech 1

    Interesting details about tech 1 implementation

    Put your most impressive technique first!

  3. Interactive Tech 2 [0:45], Interactive Tech 2 [0:45]

    Same as above

    Save these for less impressive techniques

  4. Conclusion / summary / future work [1:30]

    Summarize what you told the viewer already, and show quick clips to remind them

    End with a sentance or two on future work

    NOTE: this is just one option!! You don't have to follow it exactly.

    Example: I have a great idea for future work w/prototype
